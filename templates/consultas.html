Estoque Crítico por Armazém

Conceito: $lookup simulado — desnormalização + multi-partition scan

CQL $lookup $match
CQL (Cassandra)
SELECT produto_nome, quantidade, status
FROM estoque_por_armazem
WHERE armazem_id = ?;
MongoDB equiv
db.estoque.aggregate([
  { $match: { status: { $in: ["CRITICO","ZERADO"] } } },
  { $lookup: { from:"armazens", localField:"armazem_id",
               foreignField:"_id", as:"armazem" } },
  { $group: { _id:"$armazem_id",
      armazem: { $first:"$armazem.nome" },
      itens: { $push: "$$ROOT" } } }
])

05

UDT — Subdocumento de Endereço

Conceito: frozen<endereco> como subdocumento embutido (embedded doc)

UDT embedded doc
CQL (Cassandra)
CREATE TYPE estoque.endereco (
    logradouro text, numero text, complemento text,
    bairro text, cidade text, estado text, cep text
);

CREATE TABLE fornecedores (
    fornecedor_id uuid PRIMARY KEY,
    nome          text,
    telefones     list<text>,
    endereco      frozen<endereco>
);

SELECT endereco.cidade, endereco.estado FROM fornecedores;
MongoDB equiv
db.fornecedores.insertOne({
  nome: "Fornecedor X",
  endereco: { logradouro: "Rua Y", numero: "10", cidade: "SP" }
});
db.fornecedores.find({ "endereco.cidade": "São Paulo" })

O UDT frozen<endereco> é equivalente a um subdocumento embutido no MongoDB. A diferença é que no Cassandra o UDT frozen é serializado como blob — qualquer atualização reescreve o campo inteiro. Isso é análogo a $set: { endereco: {...} } no MongoDB (substituição do objeto completo). Para atualizações granulares por campo, seria necessário usar o UDT sem frozen (disponível em Cassandra 3.6+), que permite UPDATE … SET endereco.cidade = 'Campinas'.
